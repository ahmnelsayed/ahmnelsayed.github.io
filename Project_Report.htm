Optimal OPAMP Design

Ahmed Elsayed, Abdullah Alqarni, and Dr. Ayman Elsayed, Mentor

Abstract-Our goal for this project is to create a tool that can solve the design problem for a globally optimal design based on technology and specifications. Solving the design problem is a challenging and time-intensive part of OPAMP design. We set out to create a tool that does this in seconds. We decided to employ the trial and error method by ourselves first to build a basis for our project. The method by which we accomplish our goal is geometric programming and convex optimization. We have designed and OPAMP manually to understand the process we with to automate. We wrote, studied and understood the equations that for the design problem. We did research and found a mathematical tool called ggplab that can handle the math. We performed sweeps to find mobility multiplied by oxide capacitance. We have started writing our MATLAB code with 10 out of 22 equations working and giving results.

 

Index Terms-Computer aided engineering, Design Optimization, Integrated circuit, Integrated design, MOS integrated circuits, Operational amplifiers.

 

I.     INTRODUCTION

THIS project is about analog integrated circuit design. We are interested in the design problem. The design problem is the task of carefully choosing the design parameters which we can control to produce the desired results. In our case the design problem is choose W1, W2, . .W8, L1, L2, . .L8, I bias, Cc, Rc to get the DC Gain, Gain Bandwidth product, Common mode rejection ratio, etc. that we desire.

The goal is to automatically optimize the design parameters to meet specifications. However, the true goal of this project is to for us to learn about analog integrated circuit design.

The previous approach to solve the design problem is to change the values of W/L and bias current to find a locally optimal design. This design requires a well-chosen starting point typically from a professor or an experienced designer so that the optimal design we find is the true optimal design. The results of this is that we could only say our design is the best in the neighborhood od designs but not the best in all designs.

Using the geometric programming and convex optimization method found in our reference paper [1], we can support the claim that our design is the best design possible for the given specifications, cost function and constraints. The method we use to solve the geometric program, the set of equations, we wrote is a MATLAB toolbox called ggplab [4], which we found when researching how we could do it ourselves. The results are promising. The design parameters we are getting are within reasonable bounds for the most part, but we are still in the preliminary stages of writing the tool; since we spent a large amount of time studying to understand how we would implement it.

II.     Research and Project Outline

We set out find the best method to automatically solve the design problem. We started by looking for people who have done this before. It turns out there are many students and researchers who have created an optimization tool in some form in the past. We quickly realized that we never design an OPAMP ourselves; It would be ridiculous that try to automate a process that we have never done manually.

So, we decided that we would first design an OPAMP by hand. During our research, we found a paper authored by Thomas Lee that spoke of a superior optimization method. We were interested. We examined the claims made for this method called Geometric Programming and Convex Optimization and found that it is indeed the best method since it guarantees a Globally Optimal Design while proving infeasible requirements [1].

Unlike other methods which give inconclusive results saying this is the best this method can do but maybe some other method could do better. Geometric programming gives the best design of "any method" [1].

III.     Materials Used

There are many obstacles to design an OPAMP by hand. The main obstacle was preparing the software design environment to start the design process. We first installed a vncserver viewer. We then found the NCSU design kit and followed instruction from EECS270A_Cadence_Instructions_with_NCSU_kit.pdf [4]. We transferred the .tar files onto our eecs server. We then setup the folders for NCSU design kit in our working directory. We wrote a .cshrc setup file and put it in the home directory. We then ran cadence. After a few edits to the cds.lib file we got the device library to show up in library manager.

We then realized that the technology libraries we had only include down to 1.6-micron technology. This is far from being a relevant technology for designing an OPAMP today. So, we started looking for devices again and quickly found NCSU FreePDK45 devices, which were in 45 nanometer technology.

Then came the task of getting the simulator to run. We first attempted to work with hspice through cadence Analog Design Environment (ADE L.) After editing an error in the .include file, we were successful in running a simulation using NCSU FreePDK45 devices on hspice simulator. We were ecstatic we finally can start design. Soon after we started, we realized that these devices were very difficult to design for. For some reason they behaved unpredictably. Design with these model files was troublesome. Our mentor recommended we try a different model. We found the 45n270Amodels.sp from my Advanced Analog Integrated Circuit Design I (EECS 270A) class [5]. We were finally able to start our design with spectre as our simulator.

IV.     High-level System

After going through the design process, ourselves we found that our tool would be three parts. First the user must extract the mobility, oxide capacitance, parasitic capacitances for the given technology. Second the must input their specifications into our MATLAB variables and receive the design parameters our code using ggplab calculates. Third they must input the design values into their schematic, simulate using spectre and check if the specifications are met. Optionally they can further tweak the design values using the optimization results as starting point in order to reach a truly global design because the model we use is the square law and is not exactly tailored to their technology.
